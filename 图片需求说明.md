# 网站图片需求说明

## 图片文件夹结构
建议在项目根目录创建 `images` 文件夹，所有图片放在该文件夹下。

```
car/
├── images/
│   ├── vehicles/          # 车型图片文件夹
│   ├── logo/              # Logo图片文件夹
│   └── background/        # 背景图片文件夹（可选）
├── index.html
├── vehicles.html
├── styles.css
└── script.js
```

## 二、Logo图片（可选，但推荐）

### 路径：`images/logo/`

如果需要替换导航栏中的图标文字，可以上传Logo图片。

| 用途 | 文件名建议 | 尺寸要求 |
|---|---|---|
| 导航栏Logo | `logo.png` 或 `logo.svg` | 200x50 像素（SVG格式更佳） |
| 网站图标（Favicon） | `favicon.ico` | 32x32 或 16x16 像素 |

**注意**：目前网站使用Font Awesome图标，如果不需要Logo图片，可以保持现状。

## 图片上传后的代码修改

上传图片后，需要修改以下文件：

### 1. 修改 `script.js` 中的车型图片路径

将车型数据中的 `image` 字段从emoji改为图片路径：

```javascript
// 修改前
image: '🚗'

// 修改后
image: 'images/vehicles/byd-seal-dmi.jpg'
```

### 2. 修改 `script.js` 中的渲染函数

在 `renderVehicles` 函数中，将emoji显示改为img标签：

```javascript
// 修改前
<div class="vehicle-image">
    ${vehicle.image}
    ${hotBadge}
</div>

// 修改后
<div class="vehicle-image">
    <img src="${vehicle.image}" alt="${vehicle.brand} ${vehicle.name}" style="width: 100%; height: 100%; object-fit: cover;">
    ${hotBadge}
</div>
```

## 图片优化建议

1. **压缩图片**：使用工具如 TinyPNG、ImageOptim 压缩图片，减小文件大小
2. **WebP格式**：考虑使用WebP格式，文件更小，加载更快
3. **响应式图片**：可以为不同屏幕尺寸准备不同大小的图片
4. **懒加载**：考虑为图片添加懒加载功能，提升页面加载速度

## 优先级说明

### 必需（高优先级）
- ✅ **车型图片**：15张车型图片，直接影响车型展示页面的视觉效果

### 推荐（中优先级）
- ⭐ **Logo图片**：提升品牌识别度
- ⭐ **合作伙伴Logo**：增强专业性和可信度

### 可选（低优先级）
- 📷 **背景图片**：当前渐变背景已足够美观，背景图片为可选项

---

## 总结

**最少需要上传**：15张车型图片（放在 `images/vehicles/` 文件夹）

**推荐上传**：Logo图片 + 合作伙伴Logo（提升专业度）

**可选上传**：背景图片（当前设计已足够美观）
